## Commitment Language

### Standard Capgemini Commitments

Use these phrases when making claims:

| Claim Type | Acceptable Language |
|------------|---------------------|
| Cost savings | "Up to 25% cost savings" |
| Speed improvement | "27-50% improvement in release velocity" |
| Quality | "Less than 1% production defects" |
| Support reduction | "25-70% reduction in platform support" |
| Technical debt | "20% or more landscape reduction" (ADMnext commitment) |

### Qualifying Language

Always qualify with appropriate context:
- "Based on typical client results..."
- "Up to..."
- "Depending on scope and complexity..."
- "In comparable engagements..."

## How to Use in Proposals

### Do's
- Reference specific metrics with source
- Use qualifying language
- Tie metrics to client's specific outcomes
- Choose relevant metrics for client's situation

### Don'ts
- Guarantee specific outcomes
- Use metrics out of context
- Claim client-specific results as typical
- Make unverified claims

---

## Claim Verification Checklist

Before using a metric in a proposal:

- [ ] Metric has verified source in L1 evidence
- [ ] Metric is relevant to client's situation
- [ ] Appropriate qualifying language used
- [ ] Context provided for how metric was achieved
- [ ] Metric tied to client's stated outcomes
