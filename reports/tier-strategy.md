## Strategic Principle

**Free tier = manual process, user does the work.**
**Top tier = automated pipeline, AI does the work with rich intelligence data.**

The free tier must be genuinely useful (not a crippled demo) so users experience the value. The upgrade path should feel like removing friction, not unlocking hidden features.

## Free Tier Philosophy

The free tier is NOT a trial. It is a permanent, useful product that demonstrates value:

### What Free Users CAN Do
1. Create up to 2 proposals per month using the manual form-based flow
2. Define client challenges, outcomes, and context
3. Write proposal sections manually in the rich text editor
4. Export to DOCX format
5. Upload up to 5 reference documents
6. Manage a basic company profile with up to 3 differentiators
7. Store up to 10 evidence items

### What Free Users CANNOT Do (Upgrade Triggers)
1. **AI generation** -- They write sections manually. The "Generate Proposal" button shows an upgrade prompt.
2. **Document extraction** -- They must type context manually instead of uploading an RFP.
3. **Intelligence data** -- They research markets manually instead of using the intelligence suite.
4. **Multiple export formats** -- DOCX only. Slides, PDF, PPTX, HTML require Starter+.
5. **Semantic search** -- They browse documents manually instead of using vector search.
6. **Brand customization** -- Default styling only.

### The Upgrade Moment
The ideal upgrade trigger is when a free user:
- Has created 1-2 proposals manually and experienced the friction
- Tries to click "Generate Proposal" and sees what AI generation produces (show a preview/sample)
- Tries to upload an RFP and discovers AI extraction exists
- Realizes they need more than 5 documents in their knowledge base

**Key principle:** Show, don't gate. Free users should SEE what premium features do (previews, samples, locked UI with descriptions) rather than not knowing they exist.

## Upgrade Path Psychology

### Free -> Starter ($49/mo)
**Trigger:** "I'm tired of writing sections manually. AI can do this?"
**Value message:** "Let AI write your proposal sections. Upload RFPs and get instant extraction. 10x faster than manual."
**Key unlock:** AI generation (5 sections), document extraction, semantic search

### Starter -> Pro ($199/mo)
**Trigger:** "I want to win more. I need intelligence data and the full pipeline."
**Value message:** "Win more with procurement intelligence, bid evaluation, and quality scoring. Know your competition before you bid."
**Key unlock:** Full intelligence suite, bid evaluation, quality review, all export formats, opportunity search

### Pro -> Enterprise ($999/mo)
**Trigger:** "My team needs collaboration. We need enterprise-grade features."
**Value message:** "Scale your proposal operation. Unlimited seats, advanced analytics, competitive landscape, custom integrations."
**Key unlock:** Unlimited everything, team collaboration, win probability, competitive landscape, white-label exports

## Implementation Considerations

### What Exists Today vs. What's Needed

| Capability | Current State | Needed for Tiers |
|------------|--------------|------------------|
| Plan limits enforcement | Basic (`checkPlanLimit` in auth-api) | Expand to all feature gates |
| Usage tracking | Proposals + AI tokens | Add per-feature counters |
| Stripe integration | Single $999/mo plan | Multiple plan products |
| Feature flags | None | Need per-tier feature flags |
| UI gating | None | Need upgrade prompts at feature boundaries |
| Free tier flow | Does not exist | Need manual proposal flow without AI |

### Database Changes Needed
1. Add `plan_tier` column to `organizations` (free/starter/pro/enterprise)
2. Add `plan_limits` JSONB with per-tier maximums
3. Add `feature_flags` JSONB for per-tier feature toggles
4. Update `checkPlanLimit` to read from tier config
5. Add `upgrade_prompts` tracking for analytics on what triggers upgrades

### UI Changes Needed
1. Upgrade prompt component (reusable, shows feature preview + pricing)
2. Feature gate wrapper component (wraps premium features, shows upgrade prompt for free users)
3. Plan comparison page (replaces hardcoded $999/mo)
4. Updated settings/billing page with tier selection
5. Usage dashboard with tier-specific limits and progress bars

## Recommended Phase-In Plan

### Phase 1: Foundation (2 weeks)
- Add `plan_tier` to organizations table
- Create feature gate utility function
- Create upgrade prompt component
- Build manual proposal creation flow (for free tier: skip AI, show editor)

### Phase 2: Stripe Multi-Plan (1 week)
- Create 4 Stripe products with monthly prices
- Update checkout flow to select a plan
- Update billing page to show current plan + upgrade options
- Handle plan changes (upgrade/downgrade)

### Phase 3: Feature Gating (2 weeks)
- Gate AI generation behind Starter+
- Gate intelligence suite behind Pro+
- Gate advanced features behind Enterprise
- Add upgrade prompts at every gate point
- Add usage counters and limit enforcement

### Phase 4: Free Tier Polish (1 week)
- Ensure the manual proposal flow is genuinely useful
- Add "preview" of AI features (show what generation produces)
- Add onboarding for free users (different path than paid)
- Test the upgrade funnel end-to-end
